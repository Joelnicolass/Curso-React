Tipos de Inputs

type="text" Se utiliza para crear un campo de entrada de texto. Podemos colocar un atributo value=" " que va a ser el valor inicial del campo.
type="password" Los campos de contraseñas son muy similares a los de texto, con la diferencia de que su valor es ocultado (se reemplazan los caracteres con un símbolo) y no se permite copiarlo al portapapeles. Esto actúa como una medida de seguridad que previene que el valor sea conocido por cualquier persona que no sea el usuario.
type="email" Representa un campo para ingresar emails, si colocamos el atributo multiple podemos ingresar varias direcciones de email separadas por coma (,).
type="url" Se utiliza para ingresar direcciones url.
type="checkbox"

Intereses:

 Representa un control de dos estados que permite a los usuarios marcarlo como seleccionado o deseleccionado. El estado de una casilla de verificación es decidido por el atributo checked (si está presente, la casilla está marcada) Pero cuando el formulario es enviado, lo que se envía no es su estado de selección, sino el par nombre/valor. El valor es enviado sólo si la casilla está seleccionada. Se utiliza para preguntas que pueden tener más de una respuesta, debemos definir el valor (value) que es el valor que se envía, si no lo definimos por defecto es on (nos sirve para un botón tipo switch).
type="radio"

Selecciona un curso:

 Se utilizan para dar varias opciones al usuario y este debe seleccionar una sola. Cada uno de los botones radio pueden estar con sus propios elementos label, (cuando colocamos un elemento input adentro de un elemento label se asocian automáticamente). Los botones radio relacionados deben tener el mismo atributo name para crear un grupo y que la selección sea una. Debemos colocar un atributo value porque cuando el formulario es enviado solo la opción seleccionada es enviada junto con el valor (value) del control. Value debe ser único en cada opción.
type="hidden" Es un campo que no es visible y no se puede modificar por el usuario, pero su valor es enviado al servidor. Los controles ocultos son útiles para enviar datos al servidor definidos por el autor, basados o no en la interacción con el usuario. se utiliza para detectar bots que completan formularios de forma automática validando el campo con javascript.
type="submit" Envía el formulario al que pertenece, el atributo value="" es el nombre que muestra el boton.
type="button" Sin funcionalidad específica, representa un botón sin una acción predefinida asociada. La funcionalidad se la podemos dar con javascript.
type="reset"
Botón que restaura los contenidos de un formulario a sus valores predeterminados.
type="file"
Control que permite al usuario seleccionar y enviar un archivo. Se puede usar el atributo accept para definir los tipos de archivo que el control podrá seleccionar podemos pasar varias extensiones separadas por una coma (,) (imágenes, documentos, archivos de todo tipo). Importante al utilizar este input en un formulario debemos incluir dentro de la etiqueta "form" el atributo enctype="multipart/form-data" este codigo nos permite enviar archivos a través de un formulario.
type="image" Es un boton grafico que sirve para enviar el formulario (submit), se debe usar el atributo src para definir el origen de la imagen y el atributo alt para definir un texto alternativo. Se puede usar los atributos height y width para definir el tamaño de la imagen en píxeles.
type="color" Es un control que permite al usuario cambiar su valor de manera gráfica para seleccionar un color específico.
type="range" Se utiliza para seleccionar un numero dentro de un rango de números, en los navegadores los campos de rango son representados por algo similar a un deslizador, con los atributos min y max podemos personalizar el rango de números. Tenemos otro atributo step que limita la selección de números (step=”10” la selección de range va a ser de 10 en 10).
type="number" Se utiliza para ingresar un número, se pueden colocar atributos max y min para limitar la elección.
type="tel" Se utiliza para ingresar un número telefónico, podemos utilizar el atributo pattern para realizar validaciones o restringir las opciones.
type="date" Se utiliza para ingresar una fecha. En los navegadores modernos los campos de fecha son representados de manera gráfica (calendario).
type="datetime-local" Se utiliza para ingresar una fecha, pero esta incluye el año, mes, día y hora.
type="time" Se utiliza para ingresar una hora en especial, podemos utilizar los atributos min y max para restringir la selección.

Atributos

los atributos los podemos utilizar dentro de los inputs para restringir o ayudar al usuario a la hora de llenar un formulario

name: este asigna un nombre al control (es imprescindible para que el servidor pueda procesar el formulario). Se utiliza para hacer referencia a los datos del campo después enviar el formulario o hacer referencia al elemento en JavaScript.

value: valor inicial, en button y submit value es el nombre que aparece en el botón

disable: Podemos deshabilitar un input para que no funcione.

readonly: podemos crear un campo que sea solo lectura y que el usuario no pueda modificar:

min / max: atributos que nos limitan los valores a ingresar

minlength / maxlength atributos que limitan la cantidad de caracteres a ingresar

pattern: ayuda estableciendo expresiones regulares que cualquier valor ingresado debe acatar. Ej: input type="text" name="pass" pattern="[A-Za-z]{3}" title="solo 3 letras" required>

title: es responsable de proveer una explicación acerca de las reglas que aplican en el campo. Los navegadores pueden usar esta información para componer el mensaje de error que se muestra al usuario luego de un intento fallido de envío.

placeholder: desplega texto dentro del cuadro donde se va a ingresar texto antes de que el usuario escriba algo, esto se llama placeholder es texto que puede servir para guiar al usuario de lo que debe ingresar, se puede crear de la siguiente manera: input type="text" placeholder="ingresar texto">

required: se obliga al usuario a completar el campo antes de enviar el formulario. Se puede requerir datos específicos en un formulario y que el usuario no pueda enviar el formulario hasta que haya completado los campos de entrada. Ej: input type="text" required>
